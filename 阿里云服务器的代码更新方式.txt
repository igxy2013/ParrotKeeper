阿里云代码远程更新采用git+ssh的方式，前端采用nginx代理静态文件的方式，所以每次更新前需要现在本地用npm run build进行构建,然后再使用git push aliyun main的方式推送到阿里云服务器即可完成自动更新和重启服务。重启服务在远程服务器的git文件的钩子文件中设置了重启脚本。

'启动: sudo systemctl start parrotkeeper';
'停止: sudo systemctl stop parrotkeeper';
'重启: sudo systemctl restart parrotkeeper';
'状态: sudo systemctl status parrotkeeper';

确保启动脚本具有可执行权限
chmod +x /opt/ParrotKeeper/start_parrotkeeper.sh

重新加载并启动：
systemctl daemon-reload
systemctl start parrotkeeper.service
systemctl status parrotkeeper.service
